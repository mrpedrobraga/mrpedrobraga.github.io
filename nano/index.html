nano

nano is a strong-typed compiled language, with low level functionality but modern, elegant, intuitive syntax. You can get a feel for how the language is like to write in by looking at its keywords. Check 'em out:

if unless then else loop while for in of do break continue match when fn return let alias shared const struct class trait member await

Most importantly, nano is an art project, it was created to have a syntax and a functioning that was appealing to me specifically and is not intended to replace any programming languages out there.

Design Goals

Given it's an art project, here's the design guidelines I used to develop the language.

  • Expression

    In nano, everything is an expression. This is makes it easy to write sophisticated thoughts such as:

    obj.my_variable = (
     let a = 2
     a * 4
    )

    or this monstrosity:

    if (
     try
      func()
     catch
      print 'An error occured!'
    ) then print 'Success!'

    which leads us to the next point:

  • Freedom

    nano allows you to write it in many ways, allowing you to fully express yourself through code

    (!) Attention

    Please talk to your teammates about how you'd like to structure your code.

    There is a [nano best-practices style guide].

    Nano is also multi-paradigm, allowing you to program in a mixture of functional, object-oriented, or modular.

    It has powerful functions and useful simple data structures (such as const array and struct), for organized functional code... and also a great class and type system for your OOP needs.

  • Elegance

    A language that is easy to parse is a language that's easy to write code for (with some common sense implied). Which is why nano has a few patterns that are repeated here and there.

    For example, Nano's if statement doesn't require {}s, it operates on a single expression for the condition and a single expression for the consequence.

    if <condition> then <expression>

    The way you'd execute multiple expressions as a consequence is by using an expression grouping:

    (
     print 'Hello!'
     print 'Hello 2!'
    )

    This "grouping" can be used anywhere an expression can, so you know what you can do syntactically when you find this:

    await <expression>

    Another example of nano's syntactic elegance is that symbols often have abstract meanings they follow everywhere in the design.

    • () mean grouping, and are used to group expressions whenever. They never have any semantic implications.
    • [] mean 'collection of unnamed items' and are used for arrays, tuples, lists, sets.
    • {} mean 'collection of named items' and are used for structs, classes, traits, etc.
    • <> are related to 'template parameters' and are used to define or instantiate generic types and functions.
    • ... are related to 'filling in' and are used in ranges, collection spreading, arbitrary argument count in functions, partial function call (argument binding)... for i in 0...10 do print 'hi'

How to use

(!) You can't as of now.

Install the nano files somewhere within your PATH. Use the nano compiler nnc to compile your nano file and its dependencies and it'll generate your executable.

nnc ./file.nn

nano doesn't have a dedicated build system, instead allowing you to customize your compilation with compiler directives inside your files.
